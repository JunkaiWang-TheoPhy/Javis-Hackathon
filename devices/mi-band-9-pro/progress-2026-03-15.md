## Current Blockers

Latest observed state:

- `health_connect_ready = false`
- `bluetooth_ready = false`
- `health_connect_status = provider_update_required`
- `band_status = bluetooth_off`
- `android.permission.BLUETOOTH_CONNECT = granted=false`
- `android.permission.POST_NOTIFICATIONS = granted=false`
- no Health Connect provider package is currently visible from `pm list packages`

Latest foreground activity observed on the phone:

- `com.mi.health/com.xiaomi.fitness.access.health_connect.HealthConnectPrivacyActivity`

This means the transport and gateway code are in place, but the phone-side consent and permission flow is still incomplete.

## Manual Steps Still Required On The Phone

1. Finish the Xiaomi Fitness privacy and Health Connect consent flow.
2. Return to the `Mi Band Gateway` app.
3. Tap `Grant Android Permissions`.
4. Allow:
   - `Nearby devices`
   - notification permission
5. Tap `Grant Health Permissions`.
6. Tap `Start Gateway`.

## Expected Next-State Signal

Once the phone-side flow is complete, these values should change:

- `health_connect_ready` should become `true`
- `bluetooth_ready` should become `true`
- `band_status` should move from `bluetooth_off` to `bonded` or `connected`
- `/health/latest` should begin returning non-null `heart_rate_bpm`, `spo2_percent`, and `steps`
