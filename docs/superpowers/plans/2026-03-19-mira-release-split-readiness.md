# Mira Release Split Readiness Plan

## Goal

Tighten `Mira_Released_Version/` for repository split and public packaging.

## Steps

1. Add a release-side changelog.
2. Add repository split readiness notes.
3. Add package and license decision notes.
4. Normalize the release-side notification-router package namespace.
5. Wire these documents into the release portal and migration docs.
6. Verify package parsing, links, and release-side tests.
