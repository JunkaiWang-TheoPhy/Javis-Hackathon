# Mira Release Getting Started Design

## Goal

Add one release-side page that turns the current examples and deploy notes into a practical onboarding sequence.

## Scope

This is documentation-only. It does not add new runtime code or change deploy scripts.

## Design

The new page should sit next to `quick-start.md` under `readme/00-overview/`.

It should answer:

1. what to read first
2. what to configure next
3. how to move from core to module or service composition
4. when to use the advanced `home-stack-with-notification-router` path

`quick-start` remains the chooser page.
`getting-started` becomes the ordered operator path.
