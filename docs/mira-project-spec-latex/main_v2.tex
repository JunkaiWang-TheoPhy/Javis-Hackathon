\DeckTitle{Mira 评审说明书}
\DeckSub{proof-first judging brief | 面向评委的项目说明，而非品牌宣言}

\begin{summarybox}
\textbf{一句话定位}\par
Mira 是一个基于 OpenClaw 的主动感受型 AI 伴侣，使用眼镜、可穿戴信号与家居执行面，完成\textbf{低打扰、可解释、可控}的关怀闭环。
\end{summarybox}

\vspace{0.6em}
\begin{tabularx}{\linewidth}{|Y|Y|Y|Y|}
\hline
\rowcolor{MiraSoft}
\centering\textbf{Perception} & \centering\textbf{Memory} & \centering\textbf{Judgment} & \centering\textbf{Action}\tabularnewline
\hline
\centering 眼镜画面、心率、环境上下文 & \centering 短期状态与近期事件记忆 & \centering 判断是否值得打扰 & \centering 镜片提示、耳机/手表、家居执行\tabularnewline
\hline
\end{tabularx}

\vspace{0.6em}
\begin{accentbox}
\textbf{评委先看这三点}
\begin{tightitemize}
\item 本项目主打的不是“更复杂的 AI”，而是\textbf{更实际的 care loop}。
\item 当前材料把“已实现 / 部分实现 / 愿景”明确分层，避免把概念包装成现成能力。
\item 本次评审主打闭环是：\textbf{公开表达/演讲前支持}与\textbf{wearable -> Home Assistant} 两条已验证路径。
\end{tightitemize}
\end{accentbox}

\newpage
\section*{一、主问题与主用户}
\DeckSub{把叙事收束到一个评委最容易判断实际价值的高频时刻}

\begin{summarybox}
\textbf{主场景锚点}\par
\textbf{公开表达前的高压时刻。} 用户在上台、路演、演讲前往往不会主动对 AI 说“我现在很紧张”，但这个时刻恰恰最适合主动感知、最容易观察、也最容易形成低打扰闭环。
\end{summarybox}

\vspace{0.55em}
\begin{tabularx}{\linewidth}{Y Y}
\begin{accentbox}
\textbf{当前 AI 的问题}
\begin{tightitemize}
\item 大多依赖用户显式开口。
\item 即使会主动发消息，也常常不知道用户当下状态。
\item 在真正紧张或疲惫时，用户往往不想先发起对话。
\end{tightitemize}
\end{accentbox}
&
\begin{greenbox}
\textbf{Mira 的区别}
\begin{tightitemize}
\item 先感知，再决定是否值得行动。
\item 默认沉默；不确定时不打扰。
\item 动作不是聊天优先，而是\textbf{最小、最合适的支持}。
\end{tightitemize}
\end{greenbox}
\end{tabularx}

\vspace{0.4em}
\Metric{Current AI}{needs explicit ask}
\Metric{Mira}{acts before the ask, but only when appropriate}

\PageDivider

\textbf{为什么先锚定这一类用户？}
\begin{tightitemize}
\item 这是一个\textbf{单一、高频、容易展示}的痛点，适合 hackathon 评审。
\item 既能体现主动感知，也能体现“低打扰”和“分寸感”。
\item 其他场景如独居老人、回家关怀、儿童安全可作为\textbf{未来扩展场景}，不与 MVP 抢叙事中心。
\end{tightitemize}

\begin{bluebox}
\textbf{未来扩展方向（本次不作为主判分对象）}\par
独居老人趋势守护、下班回家照护、儿童危险姿态监测、社交记忆与长期关系理解。
\end{bluebox}

\section*{三、主 demo：演讲鼓励场景全链路}
\DeckSub{从“有感染力的故事”改成“可解释的闭环”}

\begin{summarybox}
\textbf{场景目标}\par
用户上台前不会主动说“我很紧张”，但系统可以从视觉、穿戴状态和近期上下文中判断：此时是否值得给出一句\textbf{最小支持}。
\end{summarybox}

\vspace{0.5em}
\begin{tabularx}{\linewidth}{L{0.08\linewidth}Y}
\textbf{1.} & Rokid 观察到用户已处于路演/演讲场景，周围存在 PPT 与观众等公开表达线索。\\
\textbf{2.} & 可穿戴侧出现紧张信号，例如心率提升或身体状态变化。\\
\textbf{3.} & 系统检索近期上下文与准备痕迹，形成“此刻确实值得支持”的判断。\\
\textbf{4.} & Care policy 决定是否行动。若不确定，则保持沉默。\\
\textbf{5.} & 若高置信度成立，则输出一句\textbf{最小、不过度暴露隐私}的鼓励信息。\\
\textbf{6.} & 输出优先在眼镜 / 耳机等私密界面完成，避免公共暴露与过度打扰。\\
\end{tabularx}

\vspace{0.6em}
\begin{accentbox}
\textbf{示例输出}\par
“放轻松，你肯定可以做到的。深呼吸。你这两天做了很多准备，去拿下这个舞台。”
\end{accentbox}

\PageDivider

\begin{tabularx}{\linewidth}{Y Y}
\begin{greenbox}
\textbf{这页要证明什么？}
\begin{tightitemize}
\item Mira 不是普通 chatbot。
\item 它的价值不在“能不能聊”，而在“能不能在合适时机少说一句但更有用的话”。
\item 它的关键能力是\textbf{先判断要不要行动}，而不是默认一直说话。
\end{tightitemize}
\end{greenbox}
&
\begin{bluebox}
\textbf{fallback 与边界}
\begin{tightitemize}
\item 观察不充分时，不输出支持话术。
\item 动作优先级：私密支持优于公开提醒。
\item 中高风险外发动作保留确认步骤，不在该 demo 中自动扩张。
\end{tightitemize}
\end{bluebox}
\end{tabularx}

\normalsize

\PageDivider

\begin{bluebox}
\textbf{工程语言下的四层理解}
\begin{tightitemize}
\item \textbf{Sensor ingestion:} 眼镜、可穿戴、脚本与设备事件进入系统。
\item \textbf{Memory write / retrieve:} 观察事件写入 ledger，形成可追溯上下文。
\item \textbf{Care policy engine:} 不是“看到就做”，而是先判断是否值得打扰。
\item \textbf{Action router:} 动作最终路由到眼镜、通知或 Home Assistant 执行面。
\end{tightitemize}
\end{bluebox}

\section*{六、安全与边界}
\DeckSub{把“我们重视安全”改成“当前有哪些明确规则”}

\begin{tabularx}{\linewidth}{Y Y}
\begin{bluebox}
\textbf{当前 repo 中已文件化的规则}
\begin{tightitemize}
\item 默认动作：\textbf{ask}
\item 需要已知 recipient
\item 默认要求 private channel
\item quiet hours：23:00 -- 08:00
\item rate limits：每小时 3 次、每天 10 次
\item caregiver escalation：默认 ask
\item public / social posting：block
\item secret / credential / raw private data：block
\end{tightitemize}
\end{bluebox}
&
\begin{greenbox}
\textbf{当前设计中强调的产品原则}
\begin{tightitemize}
\item 默认沉默，不确定不行动
\item 高风险外发需要确认
\item 用户持有暂停 / 擦除 / 撤销权
\item 家居执行应在 allowlist / whitelist 内进行
\item 感知是为了照顾，不是为了监控
\end{tightitemize}
\end{greenbox}
\end{tabularx}

\vspace{0.55em}
\begin{summarybox}
\textbf{为何这一页重要？}\par
Mira 的定义不是“更主动就更好”，而是\textbf{在被允许的边界内、用可审计规则主动}。这也是它与普通“全自动 agent”最关键的区别之一。
\end{summarybox}
